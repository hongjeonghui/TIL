# JavaScript 기초 문법 및 DOM 조작 정리

## 1. 변수

### 1-1. 변수 작성 규칙
- 식별자는 문자, $, _ 로 시작해야 한다.
- 대소문자 구분.
- 예약어(for, if, function 등)는 변수명으로 사용 불가.
- userName, phoneNumber처럼 의미가 드러나는 이름을 사용.
- $, _ 로 시작하는 변수는 특별한 용도로 구분하여 사용.

### 1-2. 변수명 규칙 (Naming Convention)
- camelCase: 변수, 객체, 함수
- PascalCase: 클래스, 생성자
- SNAKE_CASE: 상수(constants)
- 문법이 아니라 협업을 위한 소통 규칙이며, 가장 중요한 원칙은 일관성.

### 1-3. 변수 선언 키워드
1) let  
- 블록 스코프 지역 변수  
- 재할당 가능  
- 재선언 불가  

2) const  
- 블록 스코프 지역 변수  
- 재할당 불가  
- 재선언 불가  

3) var  
- 재할당/재선언 가능  
- 블록 스코프가 아닌 함수 스코프  
- 호이스팅 문제 때문에 사용 비권장  

### 1-4. const를 기본으로 사용하는 이유
- 변수의 목적이 명확해짐  
- 의도치 않은 값 변경 방지  
- 팀 프로젝트에서 버그 예방  

필요한 경우에만 let 사용(값이 바뀌는 변수일 때).


## 2. DOM

### 2-1. DOM의 개념
- 웹 페이지(Document)를 트리 구조로 표현한 객체 모델.
- HTML 요소, 속성, 텍스트 등이 모두 객체(Node).
- JavaScript는 DOM 객체를 통해 웹 페이지를 조작할 수 있다.

### 2-2. Document 객체
- DOM 트리의 최상위 객체.
- 페이지 구조, 요소, 텍스트에 접근하기 위한 진입점.
- 모든 요소는 document의 하위 객체로 구성됨.

예) document.title 을 이용해 <title> 변경 가능

### 2-3. DOM Tree
- HTML 요소(element)들은 부모-자식 관계로 구성.
- 각 노드는 children, parentNode 등의 프로퍼티로 연결됨.
- 핵심: 웹 요소를 객체로 제공하여 프로그래밍적으로 조작 가능하게 만든 API 구조.


## 3. DOM 선택

### 3-1. DOM 조작 기본 순서
1. 조작할 요소 선택  
2. 선택한 요소의 속성/내용/스타일 등을 조작  

### 3-2. 선택 메서드
- document.querySelector(selector)  
  - 선택자와 일치하는 첫 번째 요소 반환  
  - 없으면 null  

- document.querySelectorAll(selector)  
  - 선택자와 일치하는 모든 요소를 NodeList 형태로 반환  


## 4. DOM 조작

### 4-1. 속성(attribute) 조작

#### 클래스(class) 조작
element.classList.add("클래스명")  
element.classList.remove("클래스명")  
element.classList.toggle("클래스명")  

classList는 DOMTokenList 형태로 클래스 목록을 관리한다.

#### 일반 속성 조작
- element.getAttribute(name)  
- element.setAttribute(name, value)  
- element.removeAttribute(name)  

주의  
- getAttribute는 HTML의 초기값 기준  
- element.value 같은 프로퍼티는 현재 값 기준  
- setAttribute는 숫자, boolean을 넣어도 문자열로 변환됨

### 4-2. HTML 콘텐츠 조작
- textContent  
  - 순수 텍스트 반환 및 설정  
  - HTML 태그는 적용되지 않고 텍스트로만 처리됨

### 4-3. DOM 요소 조작

- document.createElement(tagName)  
  - 메모리에 요소 생성

- parentNode.appendChild(node)  
  - 자식으로 삽입(화면에 실제로 반영됨)

- parentNode.removeChild(node)  
  - DOM에서 제거  
  - 제거된 노드는 메모리에 남아 변수로 보관 가능

### 4-4. style 조작
- element.style.property 로 접근  
- inline style 형태로 추가됨  
- 여러 스타일을 반복해 적용해야 한다면 class로 제어하는 것이 권장됨


## 5. 오늘 학습한 주요 개념 요약

- 변수 선언 let/const의 차이와 const 기본 사용 원칙
- DOM 구조와 document 객체의 역할
- querySelector, querySelectorAll을 통한 DOM 선택
- classList, setAttribute 등으로 속성 조작
- textContent로 HTML 없는 텍스트 조작
- createElement, appendChild, removeChild를 이용한 동적 요소 생성/삭제
- style 속성 조작 방법


## 6. 느낀 점 / 추가 메모
- DOM 조작의 흐름(선택 → 조작)이 가장 중요하다는 것을 확인함.
- HTML 구조가 DOM 객체로 해석되는 방식이 시각적으로 이해됨.
- classList와 textContent의 차이, setAttribute와 프로퍼티(value 등)의 차이를 확실히 구분해야 함.
